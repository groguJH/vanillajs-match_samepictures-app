## 프로젝트 소개

해당 프로젝트는 뒤집힌 카드 중 같은 그림을 찾아 맞추는 개인 프로젝트입니다.

- 3종류의 이미지를 2장씩 배치해 총 6장의 카드로 게임을 진행할 수 있습니다.
- 카드를 클릭하면 앞면에서 뒷면으로 뒤집히는 플립 효과를 확인할 수 있습니다.
- 같은 그림을 맞추면 카드가 사라지고, 다른 그림이면 다시 뒤집힙니다.
- 모든 카드를 맞추면 축하 메시지와 사운드가 재생됩니다.
- 셔플 버튼으로 카드를 다시 섞어 새 게임처럼 초기화할 수 있습니다.
- 카드 선택, 셔플, 클리어 순간마다 각각 다른 사운드를 제공했습니다.

## 기획 의도

- JavaScript만으로 간단한 게임 로직과 상태 관리를 구현할 수 있다.
- DOM 이벤트를 활용해 카드 선택, 비교, 리셋 흐름을 제어할 수 있다.
- CSS 애니메이션과 사운드를 활용해 사용자 경험을 개선할 수 있다.

## 주요 기능

### 1. 카드 뒤집기와 매칭 판정

- 카드를 클릭하면 플립 애니메이션과 함께 이미지가 보입니다.
- 첫 번째 카드와 두 번째 카드를 비교해 같은 그림인지 판정합니다.
- 같은 그림이면 두 카드가 사라지고, 다른 그림이면 일정 시간 뒤 다시 뒤집힙니다.

### 2. 셔플과 게임 재시작

- 셔플 버튼을 누르면 카드 이미지가 다시 섞입니다.
- 게임 제목도 기본 상태로 초기화됩니다.
- 이전 선택 상태와 클리어 상태를 모두 제거한 뒤 새 게임처럼 다시 시작합니다.

### 3. 사운드와 완료 피드백

- 카드 선택 시 사운드가 재생됩니다.
- 셔플 시 별도의 사운드가 재생됩니다.
- 모든 카드를 맞추면 축하 메시지와 완료 사운드가 출력됩니다.

## 구현 포인트

### 1. 카드 선택 상태 분리

게임 진행 중에는 첫 번째 카드, 두 번째 카드, 그리고 추가 클릭 방지 상태를 각각 나누어 관리했습니다.  
이 방식으로 비교가 끝나기 전에 세 번째 카드가 선택되는 문제를 방지하고, 카드 매칭 로직을 더 안정적으로 처리할 수 있도록 구성했습니다.

### 2. 셔플 알고리즘 적용

이미지를 단순히 순서대로 배치하지 않고, 한 쌍씩 배열에 넣은 뒤 무작위로 섞어 매번 다른 보드가 나오도록 구현했습니다.  
같은 카드 게임이 반복적으로 지루해지지 않도록 하기 위해 셔플 과정을 별도 로직으로 분리했습니다.

### 3. 게임 종료 판정

매칭에 성공한 카드는 `fade-out` 클래스를 추가해 화면에서 사라지도록 처리했습니다.  
이후 전체 카드 중 사라진 카드 수를 확인해 모든 카드가 맞춰졌는지 판단하고, 게임 클리어 메시지와 사운드를 출력하도록 구현했습니다.

### 4. 사운드 재생 안정화

짧은 시간 안에 같은 사운드가 여러 번 재생될 수 있기 때문에, 재생 전에 `currentTime`을 0으로 되돌린 뒤 다시 실행하도록 구성했습니다.  
브라우저 환경에 따라 자동 재생 제한이 있을 수 있어 `catch` 처리도 함께 넣어 게임 흐름이 끊기지 않도록 했습니다.

## 배포 시 참고사항

- 별도의 환경변수 없이 정적 배포가 가능한 프로젝트입니다.
- 이미지와 사운드 파일이 함께 번들되므로 정적 호스팅 환경에서 바로 배포할 수 있습니다.
- 모바일 화면에서는 카드 레이아웃이 2열 구조로 변경되도록 구성되어 있습니다.

## 느낀 점

해당 토이 프로젝트를 통해 단순한 카드 게임도 클릭 흐름, 비교 시점, 중복 입력 방지 같은 상태 관리가 중요하다는 점을 배웠습니다.  
작은 게임이지만 UI 애니메이션과 사운드 같은 피드백 요소가 사용자 경험에 큰 영향을 준다는 점을 알게 되었습니다.
